# OOP Java Learning Repository

This repository is a beginner-friendly Java learning resource focused on object-oriented programming (OOP) and core programming practice. It contains small examples, exercises, and class-based programs that other learners can read, run, and modify while building their Java fundamentals.

## Purpose

The main goal of this repository is to support learning by example. It is meant for:

- Students practicing Java basics
- Beginners learning object-oriented programming concepts
- Anyone looking for simple Java examples to study and experiment with

## What You Will Find

This repository includes practice files covering topics such as:

- Variables and data types
- Conditionals and loops
- Methods and basic problem solving
- Classes and objects
- Encapsulation
- Inheritance
- Polymorphism
- Small lab exercises

## Project Structure

Some of the files and folders in this repository include:

- `JavaSummary.java` - a quick overview of basic Java syntax and control flow
- `Calc.java`, `prob.java`, `Time.java`, `cai.java` - standalone practice programs
- `lab/` - lab exercises and beginner OOP examples
- `lab/Inheritance/` - simple inheritance-focused examples

## Who This Is For

This repository is designed as a shared learning resource for others. If you are new to Java, you can use it to:

- Read simple examples
- Practice running Java programs
- Compare different class designs
- Improve your understanding of OOP step by step

## How To Run

Make sure Java is installed, then compile and run any file from the repository root.

Example:

```powershell
javac JavaSummary.java
java JavaSummary
```

For files inside the `lab` folder:

```powershell
cd lab
javac Main.java
java Main
```

## Learning Approach

The code in this repository is intentionally simple and educational. Some files are short demonstrations rather than full applications. The best way to use this repository is to:

1. Open a file and read through the code.
2. Compile and run it.
3. Change values or logic to observe what happens.
4. Build your own version of the example.
